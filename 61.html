Extension Doesn't Matter

The interesting thing about Node is that it assume that if you are running some file against node, file should have valid JavaScript code. Due to this extension doesn't matter. Change the extension from .js to .something and then run. 

$ node hello.something
hello, world

The disadvantage of this is, you'll not get syntax highlighting and some default support from modern text-editor if .js extension is not found.

On the other hand as it assume that the file should be a valid JavaScript, if you have .js extension you can leave off the extension.

Let's change the extension back to .js from .something and then run without extension: 

$ node hello
hello, world

Things are not ends here! Let's create a new folder greeting. Rename the hello.js file to index.js and then move this file into greeting folder. Now, if you run the following command it will execute greeting/index.js file: 

$ node greeting
hello, world

When node found that greeting is a folder and no file name is mentioned, it'll automatically run index.js.
