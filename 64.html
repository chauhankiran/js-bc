We're Not In Browser

Because windows and document are browser's objects. We're currently in terminal and due to this we don't have access to those objects. Instead we have other objects that do many useful things on terminal and system side such as process, os, fs, and many others.

Let's try process.pid. Process is an object in Node and pid is content of this object. This will return a process id of process in execution. We can print this using console.log. Create a new file with name pid.js and type following code:

console.log(process.pid)

You know how to run this program, right? 

$ node pid.js
2585

That's okay, if you get different process id, it is based on system you are running.
