require()

Back to our type.js program, in order to resolve the os, we need to include it in our type.js program. The inclusion of os can be done using require() function. require() function is part of globals and due to this, it is automatically available for you to use. As mentioned on docs, it take one argument called the id as string and return the content of that module. id is just a short string for each module written already in documentation such as process, console, os, and so on.

When we require os using require() function it return the content of os such as methods and objects. The recommendation from community is to save this content into a object variable and then use that object variable to call the needed methods or constants from os in following way. 

const os = require('os')

os.type()

First we require() this os module by passing the string id of it and then saved the return content into os constant. Now, I can access all the possible methods and constants available in os module. To try out this, I just call os.type().

Due to this require() now os is defined within our program. So, let's add this line in our type.js program at the top of our file because always inclusion happen at the top of the file. 

const os = require('os')

os.type()

Now, running this program will print following output: 

$ node type.js
Linux

For me it returns Linux as I've GNU/Linux system.
